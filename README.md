简单介绍核心处理流程
词法分析：程序从左到右逐个字符扫描输入表达式。它会自动跳过空格，识别并读取完整的数字（包括小数），提取函数名（如 sin, cos），并区分运算符和括号。
栈管理：使用两个栈来协同工作。
操作数栈：用于存放解析出的数字以及中间计算结果。
运算符栈：用于存放运算符（如+, -, *, /）、括号以及待处理的函数名。
优先级与结合性处理：算法为每个运算符定义了优先级（例如，乘除高于加减）。当遇到新的运算符时，会将其与运算符栈顶的符号比较优先级。如果栈顶运算符优先级更高或相等（且为左结合），则先将栈顶运算符弹出并进行计算，确保高优先级的运算先执行。这对于幂运算^这类右结合运算符有特殊处理。
特殊元素处理：
括号：左括号(直接入栈，作为子表达式的开始标记。遇到右括号)时，持续弹出运算符并进行计算，直到遇到匹配的左括号，从而完成括号内子表达式的求值。
函数：当识别到函数名时，会将其压入一个专门的函数栈。在后续遇到右括号且运算符栈顶为对应左括号时，表明函数参数已计算完毕，此时会应用该函数到其参数上。
一元运算符（正负号）：程序能区分表示正负的一元运算符+/-和表示减法的二元运算符。在表达式开头或左括号、逗号后出现的+/-会被视为一元运算符，并被特殊标记（如p代表正，n代表负）并入栈，它们具有较高的优先级。

我认为这个程序最大的难点就在于优先级的实现。而栈简直就是为实现优先级量身定制的法宝。
